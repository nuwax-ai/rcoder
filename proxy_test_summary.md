## 关键改进点

1. **路径参数优先**: 端口提取逻辑优先从路径中获取，然后才检查查询参数
2. **简化查询参数处理**: 新实现中不再手动添加 `port` 参数到查询字符串
3. **更好的路径处理**: 改进了路径前缀移除和重组逻辑
4. **向后兼容性**: 保留了原有的查询参数方式

## 使用示例

```bash
# 新的路径参数方式
curl http://localhost:3000/proxy/8766/api/users?status=active

# 简单端口方式
curl http://localhost:3000/proxy/8766

# 向后兼容的查询参数方式
curl "http://localhost:3000/proxy?port=8766&path=/api/users"
```

所有这些请求都会被正确解析并转发到目标服务 `http://127.0.0.1:8766/api/users?status=active`
