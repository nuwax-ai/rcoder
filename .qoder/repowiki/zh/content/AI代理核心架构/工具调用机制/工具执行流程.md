## 工具调用事件流的创建与状态流转

工具执行的生命周期始于 `ToolCallEventStream` 的创建，该结构体封装了工具调用过程中的事件推送能力。每个工具调用实例通过唯一的 `tool_use_id` 标识，并绑定到一个 `ThreadEventStream` 用于向客户端广播事件。

工具调用的状态流转遵循以下生命周期：
- **Pending（等待）**：当模型输出 `ToolUse` 事件时，系统创建 `ToolCallEventStream` 并发送 `send_tool_call` 事件，此时工具状态为 `WaitingForConfirmation` 或直接进入 `InProgress`。
- **InProgress（执行中）**：调用 `run` 方法启动异步任务，状态更新为 `InProgress`，并通过 `update_fields` 推送中间内容。
- **Completed（完成）**：任务成功完成后，状态设为 `Completed`，并携带 `raw_output`；若失败则设为 `Failed`。

状态变更通过 `acp::ToolCallStatus` 枚举管理，并由 `ThreadEventStream` 实时广播至前端。

## 工具调度机制execute_tool_call 方法解析

工具调度的核心逻辑位于 `handle_tool_use_event` 中，该方法响应来自语言模型的 `ToolUse` 事件。其主要职责包括：
1. 查找注册的工具实现：优先从本地工具注册表查找，若未找到则尝试从上下文服务器（Context Server）中获取。
2. 创建 `ToolCallEventStream`：用于后续事件推送。
3. 验证输入完整性：若 `is_input_complete` 为假，则暂停执行。
4. 调用 `run` 方法启动工具执行。

工具查找失败时，系统会立即返回一个失败状态的 `ToolCall`，包含错误信息 `"No tool named {} exists"`。

## 异步执行上下文构建以 edit_file_tool 和 web_search_tool 为例

### edit_file_tool
`EditFileTool` 在执行时需访问项目文件系统和编辑上下文。它通过 `authorize` 方法请求用户授权，并在 `run` 方法中使用 `background_spawn` 启动异步任务。任务内部调用编辑代理完成文本替换，并生成 `diff` 输出。

权限控制通过全局设置 `always_allow_tool_actions` 决定是否跳过授权。

### web_search_tool
`WebSearchTool` 使用 `cx.background_spawn` 创建异步任务，首先检查当前激活的搜索引擎提供商，然后发起搜索请求。执行过程中通过 `emit_update` 函数分阶段更新结果：
- 初始标题为 `"Searching the Web"`
- 搜索完成后更新为 `"Searched the web: X results"`
- 结果以 `ResourceLink` 形式嵌入 `content` 字段

两种工具均通过 `event_stream` 实现进度反馈，确保客户端能实时感知执行状态。

## 执行进度与中间结果的实时推送机制

工具执行过程中，通过 `ToolCallEventStream` 的 `update_fields` 方法向客户端推送中间状态。典型更新字段包括：
- `title`：反映当前操作阶段
- `content`：包含中间输出（如搜索结果、思考日志）
- `status`：状态变更（如 `InProgress` → `Completed`）

事件最终通过 `ThreadEventStream` 的 `unbounded_send` 发送为 `ThreadEvent::ToolCallUpdate`，由前端监听并更新 UI。

测试代码验证了此类更新的正确性，例如 `expect_tool_call_update_fields` 断言标题和内容的预期值。

## 错误传播机制与超时控制

错误处理贯穿整个执行链路：
- 工具未找到：返回 `Failed` 状态并附带错误消息
- 权限拒绝：`authorize` 返回 `Err("Permission to run tool denied by user")`
- 执行异常：异步任务捕获错误并通过 `event_stream.send_error` 上报

系统未显式实现超时控制，依赖底层服务（如 HTTP 客户端、上下文服务器）的超时配置。错误信息通过 `ThreadEvent::Error` 事件传递，确保客户端可展示详细错误。

## 执行上下文的传递项目根路径与权限上下文

执行上下文通过以下方式传递：
- **文件系统访问**：`ToolCallEventStream` 可选携带 `fs: Option<Arc<dyn Fs>>`，供工具进行文件读写。
- **项目上下文**：`EditFileTool` 持有 `project: Entity<Project>`，用于解析路径和语言特性。
- **权限上下文**：通过 `cx: &mut App` 访问全局设置（如 `AgentSettings`），决定是否自动授权。
- **语言注册表**：`language_registry` 用于语法高亮和代码分析。

上下文在工具实例化时注入，确保执行环境具备必要资源。

```mermaid
sequenceDiagram
participant Model as 语言模型
participant Thread as Agent线程
participant Tool as 工具实现
participant Stream as ToolCallEventStream
participant Client as 客户端
Model->>Thread : ToolUse事件
Thread->>Thread : 创建ToolCallEventStream
Thread->>Stream : send_tool_call()
Client<-Stream : 接收Pending状态
Thread->>Tool : 调用run()
Tool->>Stream : authorize()
Stream-->>Client : 请求用户授权
Client->>Stream : 授权响应
Stream->>Tool : 授权通过

Stream-->>Client : 推送中间结果
Tool->>Stream : 完成执行
Stream->>Thread : 返回结果
Thread->>Stream : update_fields(Completed)
Stream-->>Client : 推送完成状态
```
