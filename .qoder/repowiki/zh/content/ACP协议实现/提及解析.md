## 简介
提及解析模块实现了@提及语法的完整解析功能，能够识别和处理用户输入中的各种实体引用。该系统通过统一的URI方案解析文件路径、函数名、线程、规则等上下文元素，构建抽象语法树并进行语义验证，最终将解析结果用于激活相应工具或触发上下文检索操作。

## 提及解析器架构
提及解析器采用分层架构设计，包含词法分析、语法解析和语义验证三个主要阶段。解析器以URI格式作为输入，通过模式匹配识别不同类型的提及，并将其转换为结构化的MentionUri枚举类型。

## 词法分析与URI解析
词法分析阶段负责将原始输入字符串分解为有意义的词法单元。解析器使用url crate进行基础的URI解析，然后根据scheme和路径信息确定提及类型。

```mermaid
flowchart TD
Start([输入字符串]) --> ParseURL["解析为URL对象"]
ParseURL --> GetScheme["提取scheme"]
GetScheme --> SchemeCheck{"scheme检查"}
SchemeCheck --> |file| HandleFile["处理文件URI"]
SchemeCheck --> |zed| HandleZed["处理zed URI"]
SchemeCheck --> |http/https| HandleHttp["处理HTTP URI"]
SchemeCheck --> |其他| Error["错误: 未识别的scheme"]
HandleFile --> FilePath["提取文件路径"]
HandleFile --> FragmentCheck{"存在fragment?"}
FragmentCheck --> |是| ParseLineRange["解析行号范围"]
FragmentCheck --> |否| CheckDirectory["检查是否为目录"]
ParseLineRange --> QueryCheck{"存在symbol查询参数?"}
QueryCheck --> |是| CreateSymbol["创建Symbol提及"]
QueryCheck --> |否| CreateSelection["创建Selection提及"]
CheckDirectory --> |以/结尾| CreateDirectory["创建Directory提及"]
CheckDirectory --> |否则| CreateFile["创建File提及"]
HandleZed --> ZedPathCheck{"路径前缀检查"}
ZedPathCheck --> |/agent/thread/| CreateThread["创建Thread提及"]
ZedPathCheck --> |/agent/text-thread/| CreateTextThread["创建TextThread提及"]
ZedPathCheck --> |/agent/rule/| CreateRule["创建Rule提及"]
ZedPathCheck --> |/agent/pasted-image| CreatePastedImage["创建PastedImage提及"]
ZedPathCheck --> |/agent/untitled-buffer| CreateUntitledSelection["创建无标题缓冲区Selection"]
HandleHttp --> CreateFetch["创建Fetch提及"]
```

## 正则表达式模式与解析示例
系统使用结构化的模式而非正则表达式进行解析，但其模式匹配逻辑可以类比为正则表达式。

### 文件提及模式
```
file://<绝对路径>(#L<起始行>:<结束行>)?
```

### 符号提及模式
```
file://<绝对路径>\?symbol=<符号名>#L<起始行>:<结束行>
```

### 线程提及模式
```
zed:///agent/thread/<会话ID>\?name=<线程名>
```
