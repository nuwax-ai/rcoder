## 简介

本文档旨在为新加入项目的开发者提供详细的本地开发环境搭建指南。RCoder 是一个基于 Rust 构建的现代化 AI 驱动开发平台，通过 ACP (Agent Client Protocol) 协议实现与多种 AI 代理的统一交互。本指南将指导您完成从 Rust 工具链安装到服务运行验证的完整流程。

## Rust工具链安装与配置

### 安装rustup

`rustup` 是 Rust 的官方工具链管理器，用于安装和管理不同版本的 Rust 编译器。

```bash
# 下载并安装 rustup
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 或者使用国内镜像加速
curl -L https://rsproxy.cn/install.sh | sh
```

安装完成后，重新加载 shell 配置或重启终端。

### 安装核心工具

安装 Rust 编译器、包管理器和代码格式化工具：

```bash
# 安装稳定版 Rust 工具链
rustup install stable

# 设置为默认工具链
rustup default stable

# 安装 cargo（包管理器，通常随 rustup 自动安装）
rustup component add cargo

# 安装 rustfmt（代码格式化工具）
rustup component add rustfmt

# 安装 clippy（代码检查工具）
rustup component add clippy
```

### 验证安装

```bash
# 验证 Rust 编译器版本
rustc --version

# 验证 Cargo 版本
cargo --version

# 验证 rustfmt
cargo fmt --version

# 验证 clippy
cargo clippy --version
```

### IDE集成（推荐使用rust-analyzer）

为了获得最佳的开发体验，推荐使用 `rust-analyzer` 作为 IDE 的 Rust 语言服务器。

**VS Code 配置：**
1. 安装 "Rust Analyzer" 扩展
2. 确保 `rust-analyzer` 在系统 PATH 中可用
3. 打开项目文件夹，rust-analyzer 将自动激活

**其他编辑器：**
- **IntelliJ IDEA**: 安装 "Rust" 插件
- **Vim/Neovim**: 配置 `coc-rust-analyzer` 或 `rust-tools.nvim`
- **Emacs**: 配置 `eglot` 或 `lsp-mode` 与 rust-analyzer

## 项目克隆与工作区初始化

### 克隆仓库

```bash
# 克隆项目仓库
git clone https://github.com/your-org/rcoder.git
cd rcoder
```

### 初始化Workspace

本项目采用 Cargo Workspace 结构，包含多个独立的 crate。

```bash
# 查看 workspace 结构
cat Cargo.toml

# 初始化 workspace
cargo init --vcs git

# 构建整个 workspace
cargo build --workspace
```

### 验证项目结构

项目主要包含以下核心组件：
- `crates/rcoder`: 主应用（Axum 路由、业务逻辑）
- `crates/pingora-proxy`: Pingora 反向代理封装
- `crates/codex-acp-agent`: Codex AI 代理实现
- `crates/claude-code-agent`: Claude Code AI 代理实现
- `crates/nuwax_parser`: 项目文件解析器

## 依赖安装与环境变量配置

### 安装项目依赖

```bash
# 构建所有 workspace 成员
cargo build --workspace

# 或者只构建主应用
cargo build --bin rcoder
```

### 安装AI代理依赖

根据项目需求，安装相应的 AI 代理 CLI 工具。

```bash
# 安装 Claude Code CLI (npm 方式)
npm install -g @anthropic-ai/claude-code

# 或者使用 pip 方式
pip install claude-dev
```

### 环境变量配置

创建 `.env` 文件配置必要的环境变量：

```env
# 日志级别
RUST_LOG=debug

# 数据库连接（可选）
DATABASE_URL=sqlite:///./rcoder.db

# Claude Code CLI 路径（可选）
CLAUDE_CODE_PATH=claude

# RCoder 服务端口（覆盖配置文件）
RCODER_PORT=3000
```

### 配置文件说明

项目使用 `config.yml` 进行配置，首次启动时会自动生成默认配置文件。

```yaml
# rcoder 配置文件
default_agent: Codex
projects_dir: ./project_workspace
port: 3000
proxy_config:
  listen_port: 8080
  default_backend_port: 3000
  backend_host: "127.0.0.1"
  port_param: "port"
```

**配置优先级：** 命令行参数 > 环境变量 > 配置文件 > 默认配置

## 服务运行与测试

### 运行主服务

```bash
# 启动主服务（Axum），默认端口 3000
cargo run --bin rcoder

# 指定端口和项目目录
cargo run --bin rcoder -- --port 8087 --projects-dir ./my-projects
```

### 启用代理服务

```bash
# 启用 Pingora 反向代理，监听端口 8080
cargo run --bin rcoder -- --enable-proxy --proxy-port 8080

# 指定默认后端端口
cargo run --bin rcoder -- --enable-proxy --proxy-port 8080 --default-backend-port 3000
```

### 运行测试用例

```bash
# 运行所有测试
cargo test

# 运行特定包的测试
cargo test --package rcoder

# 运行集成测试
cargo test --test integration

# 代码格式化检查
cargo fmt -- --check

# 代码质量检查
cargo clippy -- -D warnings
```

## 常见问题排查

### 编译错误

**问题：** `cargo build` 失败，出现依赖解析错误

**解决方案：**
```bash
# 清理构建缓存
cargo clean

# 更新依赖锁文件
cargo update

# 重新构建
cargo build --workspace
```

### 依赖冲突

**问题：** 不同 crate 依赖同一库的不同版本

**解决方案：**
1. 检查 `Cargo.lock` 中的版本冲突
2. 在 `Cargo.toml` 中明确指定兼容版本
3. 使用 `cargo tree` 分析依赖树

```bash
# 查看依赖树
cargo tree | grep <package-name>
```

### 端口占用

**问题：** `address already in use` 错误

**解决方案：**
```bash
# 查找占用端口的进程
lsof -i :3000
# 或
netstat -tulpn | grep :3000

# 终止占用进程
kill <PID>

# 或使用其他端口
cargo run --bin rcoder -- --port 8087
```

### AI代理连接失败

**问题：** Claude Code 或 Codex 代理无法连接

**解决方案：**
1. 确认 CLI 工具已正确安装
2. 检查 API 密钥环境变量
   ```bash
   export ANTHROPIC_API_KEY="your-api-key"
   ```
3. 验证网络连接和代理设置

### 配置文件错误

**问题：** YAML 配置文件解析失败

**解决方案：**
1. 检查 YAML 语法是否正确
2. 确保缩进一致（使用空格，非 Tab）
3. 验证字段名称拼写

```bash
# 使用在线 YAML 验证器检查
# 或使用命令行工具
yamllint config.yml
```

### 命令行参数参考

| 参数 | 短参数 | 说明 |
|------|--------|------|
| `--port` | `-p` | 设置主服务端口（Axum） |
| `--projects-dir` | `-d` | 设置项目工作目录 |
| `--enable-proxy` | 无 | 启用 Pingora 反向代理 |
| `--proxy-port` | 无 | 设置 Pingora 监听端口 |
| `--default-backend-port` | 无 | 未指定端口时的默认后端端口 |
