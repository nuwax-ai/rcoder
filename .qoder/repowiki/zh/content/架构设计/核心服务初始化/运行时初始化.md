## 核心组件

rcoder应用的核心组件围绕Tokio异步运行时构建，主要包括主应用服务、代理服务管理和清理任务三大模块。主应用通过Axum框架提供HTTP接口，代理服务处理AI代理的生命周期，清理任务负责回收闲置资源。

## 运行时初始化流程

rcoder应用的运行时初始化流程始于`main.rs`文件中的`#[tokio::main]`宏。该宏启动一个多线程Tokio运行时，执行异步的`main`函数。初始化过程包括遥测系统设置、配置加载、工作目录创建和代理服务启动等关键步骤。

```mermaid
flowchart TD
Start([应用启动]) --> InitTelemetry["初始化遥测系统"]
InitTelemetry --> ParseArgs["解析命令行参数"]
ParseArgs --> LoadConfig["加载配置文件"]
LoadConfig --> CreateDir["创建项目工作目录"]
CreateDir --> CreateChannel["创建本地任务通道"]
CreateChannel --> StartLocalRuntime["启动单线程LocalSet运行时"]
StartLocalRuntime --> StartProxy["启动代理服务"]
StartProxy --> CreateAppState["创建应用状态"]
CreateAppState --> CreateRouter["创建路由"]
CreateRouter --> StartServer["启动HTTP服务器"]
StartServer --> AwaitProxy["等待代理服务完成"]
AwaitProxy --> End([初始化完成])
```

## 多线程Tokio运行时配置

主应用使用`#[tokio::main]`宏配置多线程Tokio运行时，该宏在编译时生成适当的运行时初始化代码。多线程运行时默认启用所有功能（I/O、定时器等），并根据系统核心数自动配置工作线程。

## 单线程LocalSet设计

为处理非Send类型的AI代理worker，应用创建了一个独立的单线程Tokio运行时，并在其上运行`LocalSet`。这种设计允许在特定线程上执行不能跨线程移动的任务，确保AI代理worker的稳定运行。

```mermaid
sequenceDiagram
participant MainThread as "主线程"
participant LocalThread as "本地线程"
participant LocalSet as "LocalSet"
participant AgentWorker as "Agent Worker"
MainThread->>LocalThread : spawn(闭包)
LocalThread->>LocalThread : 创建单线程运行时
LocalThread->>LocalSet : 创建LocalSet
LocalSet->>AgentWorker : run_until(agent_worker)
AgentWorker->>AgentWorker : 处理LocalSetAgentRequest
AgentWorker->>MainThread : 通过通道通信
```

## 服务初始化与任务管理

应用通过通道机制在不同运行时之间通信。主运行时通过`unbounded_channel`向LocalSet中的agent worker发送请求，实现了跨运行时的任务调度。这种设计分离了HTTP服务和代理服务的关注点。

```mermaid
flowchart LR
A[HTTP请求] --> B[主Tokio运行时]
B --> C[AppState]
C --> D[通过local_task_sender发送]
D --> E[LocalSet运行时]
E --> F[agent_worker处理]
F --> G[执行AI代理任务]
G --> H[通过SSE返回进度]
```

## 异步任务生命周期管理

应用使用`JoinHandle`管理异步任务的生命周期。代理服务通过`tokio::spawn`启动，并返回`JoinHandle`用于等待任务完成。清理任务使用`spawn_local`在LocalSet中运行，确保与特定运行时绑定。

```mermaid
stateDiagram-v2
[*] --> TaskCreated
TaskCreated --> TaskRunning : spawn()
TaskRunning --> TaskCompleted : 正常完成
TaskRunning --> TaskCancelled : cancel()
TaskRunning --> TaskFailed : 发生错误
TaskCompleted --> [*]
TaskCancelled --> [*]
TaskFailed --> [*]
```
