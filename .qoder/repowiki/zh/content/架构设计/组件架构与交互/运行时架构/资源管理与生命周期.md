系统采用RAII（Resource Acquisition Is Initialization）原则实现资源管理，核心是`AgentLifecycleGuard`结构体。当代理实例被创建时，会生成一个生命周期守卫对象，该对象在被丢弃（drop）时自动执行资源清理。

## 会话缓存设计与线程安全

### 全局会话缓存架构
系统使用`LazyLock`初始化全局`DashMap`来管理会话缓存，实现了按`session_id`分组的统一会话消息缓存。每个会话的数据由`SessionData`结构体包装，内部使用`ringbuf`实现的循环缓冲区存储消息。

### 线程安全实现机制
会话缓存系统通过多层并发控制确保线程安全：
1. 外层使用`DashMap`提供分片锁机制，减少锁竞争
2. 内层使用`Mutex`保护`HeapRb`循环缓冲区的访问
3. `LazyLock`确保全局静态变量的线程安全初始化

```mermaid
flowchart TD
Start([开始添加消息]) --> LockDashMap["获取DashMap读/写锁"]
LockDashMap --> CheckExist{"会话是否存在?"}
CheckExist --> |否| CreateSession["创建新SessionData(1000条)"]
CheckExist --> |是| GetSession["获取现有SessionData"]
CreateSession --> InsertSession["插入DashMap"]
GetSession --> InsertSession
InsertSession --> LockMutex["获取rb Mutex"]
LockMutex --> PushMessage["push_overwrite消息"]
PushMessage --> UpdateCount["更新消息计数"]
UpdateCount --> ReleaseMutex["释放Mutex"]
ReleaseMutex --> End([结束])
```

- `drain_messages`: 获取所有消息并清空缓存，用于SSE初始推送

### 清理执行逻辑
清理任务的核心逻辑在`cleanup_idle_agents`方法中实现，遵循以下步骤：
1. 获取当前时间并统计活动代理数量
2. 遍历`PROJECT_AND_AGENT_INFO_MAP`，识别闲置超时的代理
3. 对每个闲置代理执行RAII清理
4. 更新统计信息并记录日志


## 实际应用场景分析

### 会话超时处理流程
当用户长时间不活动时，系统会自动清理闲置的代理实例。这一过程由后台清理任务定期触发，确保系统资源的有效利用。

```mermaid
sequenceDiagram
participant Task as 清理任务
participant Map as PROJECT_AND_AGENT_INFO_MAP
participant Guard as AgentLifecycleGuard
participant Process as 子进程/IO任务
Task->>Task : tick(5分钟间隔)
Task->>Map : 遍历所有代理
Map->>Task : 返回Idle状态代理
Task->>Task : 检查最后活动时间
alt 超过30分钟
Task->>Map : remove(project_id)
Map->>Guard : Drop
Guard->>Guard : 发送取消信号
Guard->>Process : 终止/取消
Guard->>Task : 资源清理完成
Task->>Task : 更新统计
else 未超时
Task->>Task : 忽略
end
```

### 异常终止资源回收
当系统异常终止或代理服务崩溃时，`AgentLifecycleGuard`的`Drop`实现确保了资源的可靠回收。即使在异常情况下，关键资源如子进程和异步任务也会被正确清理。

```mermaid

E --> F["标记为已停止"]
F --> G[资源完全释放]
C --> |否| H[忽略(仍有其他引用)]
```

## 结论
本系统通过精心设计的生命周期管理策略，实现了AI代理实例、会话缓存和系统资源的高效管理。核心优势包括：
1. **RAII资源管理**: 通过`AgentLifecycleGuard`确保资源的自动清理，避免资源泄漏
2. **线程安全缓存**: 使用`DashMap`和`ringbuf`实现高性能、线程安全的会话消息缓存
3. **智能清理机制**: 基于闲置超时的后台清理任务，保障长时间运行的内存稳定性
4. **优雅停止**: 支持通过API请求或自动超时机制优雅地停止代理服务

这些设计共同确保了系统在高并发场景下的稳定性和可靠性，为AI代理服务提供了坚实的基础设施支持。
