## 简介
RCoder 是一个基于 Rust 构建的现代化 AI 驱动开发平台，通过 ACP (Agent Client Protocol) 协议实现与多种 AI 代理的统一交互。本快速入门指南旨在帮助新手用户在本地环境中快速部署和运行 RCoder 服务，完成从零到第一个 AI 请求的完整流程。

## 环境要求
在开始部署 RCoder 服务之前，请确保您的系统满足以下基本要求：

- **Rust 版本**: 1.75 或更高版本（推荐使用 2024 Edition）
- **构建工具**: Cargo（随 Rust 自动安装）
- **可选依赖**:
  - Claude Code CLI（用于 Claude 代理）
  - OpenAI Codex（用于 Codex 代理）
- **操作系统**: 支持 Linux、macOS 和 Windows

您可以通过以下命令检查 Rust 和 Cargo 的安装情况：
```bash
rustc --version
cargo --version
```

如果尚未安装 Rust，建议使用 rustup 进行安装：
```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

## 部署与运行
本节将指导您完成 RCoder 服务的部署和启动过程。

### 1. 克隆并构建项目
首先，克隆 RCoder 项目仓库并进入项目目录：
```bash
git clone https://github.com/your-org/rcoder.git
cd rcoder
```

然后使用 Cargo 构建整个工作区：
```bash
cargo build --workspace
```

### 2. 启动主服务
启动 RCoder 主服务（基于 Axum 框架）：
```bash
# 使用默认端口 3000
cargo run --bin rcoder

# 指定自定义端口和项目目录
cargo run --bin rcoder -- --port 8087 --projects-dir ./my-projects
```

### 3. 启用反向代理
要启用 Pingora 反向代理功能，请使用以下命令：
```bash
# 启用代理并指定监听端口
cargo run --bin rcoder -- --enable-proxy --proxy-port 8080

# 指定默认后端端口
cargo run --bin rcoder -- --enable-proxy --proxy-port 8080 --default-backend-port 3000
```

启动成功后，您将在日志中看到类似以下信息：
```
启动 Pingora 反向代理服务，监听端口: 8080
📡 监听端口: 8080
🔄 路由规则: /proxy/{port}/{path}
```

## 配置文件详解
RCoder 支持通过 `config.yml` 文件进行配置，该文件在首次启动时会自动生成。

### 配置优先级
RCoder 的配置优先级从高到低为：
1. **命令行参数** - 最高优先级
2. **环境变量** - 中等优先级
3. **配置文件** - 较低优先级
4. **默认配置** - 最低优先级

### config.yml 结构
以下是 `config.yml` 文件的完整结构和关键参数说明：

```yaml
# rcoder 配置文件
# 该文件在首次启动时自动生成

# 默认使用的 AI 代理类型 (Codex/Claude/Proxy)
default_agent: Codex

# 项目工作目录
projects_dir: ./project_workspace

# 主服务端口
port: 3000

# Pingora 反向代理配置
proxy_config:
  # 代理服务监听端口 (用于接收外部请求)
  listen_port: 8080
  # 默认后端服务端口 (当请求未指定端口时使用)
  default_backend_port: 3000
  # 后端服务主机地址
  backend_host: "127.0.0.1"
  # URL 中端口参数的名称 (用于从路径中提取端口号)
  port_param: "port"
```

### 命令行参数覆盖
您可以通过命令行参数覆盖配置文件中的设置，这些参数具有最高优先级：

| 参数 | 短参数 | 说明 | 示例 |
|------|--------|------|------|
| `--port` | `-p` | 设置主服务端口（Axum） | `--port 8087` |
| `--projects-dir` | `-d` | 设置项目工作目录 | `--projects-dir ./projects` |
| `--enable-proxy` | 无 | 启用 Pingora 反向代理 | `--enable-proxy` |
| `--proxy-port` | 无 | 设置 Pingora 监听端口 | `--proxy-port 8080` |
| `--default-backend-port` | 无 | 未指定端口时的默认后端端口 | `--default-backend-port 3000` |

查看所有可用参数：
```bash
cargo run --bin rcoder -- --help
```

## 调用聊天API
本节将演示如何通过 curl 命令调用 RCoder 的聊天 API 接口，并接收 SSE 格式的响应流。

### 1. 健康检查
首先，验证服务是否正常运行：
```bash
curl -X GET http://localhost:3000/health
```

预期响应：
```json
{
  "status": "ok",
  "timestamp": "2024-01-01T00:00:00Z"
}
```

### 2. 发送聊天请求
向 AI 代理发送聊天消息：
```bash
curl -X POST http://localhost:3000/chat \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "你好，请帮我创建一个 Rust Web API 项目",
    "project_id": "my_rust_project"
  }'
```

成功响应示例：
```json
{
  "success": true,
  "data": {
    "project_id": "my_rust_project",
    "session_id": "session456",
    "error": null
  },
  "error": null
}
```

### 3. 接收SSE响应流
使用 Server-Sent Events (SSE) 协议接收实时进度更新：
```bash
curl -X GET http://localhost:3000/agent/progress/your-session-id \
  -H "Accept: text/event-stream"
```

SSE 事件格式示例：
```
event: prompt_start
data: {"session_id":"session456","message_type":"SessionPromptStart","sub_type":"prompt_start","data":{"prompt":"你好，请帮我创建一个 Rust Web API 项目"},"timestamp":"2024-01-01T00:00:00Z"}

event: agent_message_chunk
data: {"session_id":"session456","message_type":"AgentSessionUpdate","sub_type":"agent_message_chunk","data":{"content":{"type":"text","text":"正在为您创建 Rust Web API 项目..."}},"timestamp":"2024-01-01T00:00:01Z"}

event: prompt_end
data: {"session_id":"session456","message_type":"SessionPromptEnd","sub_type":"end_turn","data":{"stop_reason":"end_turn","message":"项目创建完成"},"timestamp":"2024-01-01T00:00:05Z"}
```

## 测试反向代理功能
RCoder 提供了 `test_proxy.sh` 脚本来验证反向代理功能。

### 1. 运行测试脚本
```bash
# 确保有执行权限
chmod +x test_proxy.sh

# 运行测试
./test_proxy.sh
```

### 2. 测试内容
该脚本将执行以下测试步骤：
1. 启动两个测试后端服务（端口 3000 和 3001）
2. 启动 RCoder 代理服务（监听端口 8080）
3. 测试不同方式的代理请求：
   - 查询参数方式代理到端口 3000
   - 查询参数方式代理到端口 3001
   - 路径方式代理到端口 3000
   - 默认端口代理

### 3. 手动测试示例
您也可以手动测试代理功能：
```bash
# 假设本机有服务运行在 5173 端口
curl "http://127.0.0.1:8080/proxy/5173/page/1977625137029189632/prod/"
```

如果请求被误发到主服务端口，可以使用 `--location` 参数自动跟随重定向：
```bash
curl --location "http://127.0.0.1:3000/proxy/5173/page/1977625137029189632/prod/"
```

## 完整操作流程示例
现在，让我们通过一个完整的操作流程来体验 RCoder 的全部功能。

### 步骤 1: 准备环境
```bash
# 克隆项目
git clone https://github.com/your-org/rcoder.git
cd rcoder

# 构建项目
cargo build --workspace
```

### 步骤 2: 启动服务
```bash
# 启动主服务并启用代理
cargo run --bin rcoder -- --port 8087 --enable-proxy --proxy-port 8080 --projects-dir ./my_projects
```

### 步骤 3: 发送聊天请求
```bash
# 发送请求并获取 session_id
RESPONSE=$(curl -s -X POST http://localhost:8087/chat \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "请帮我创建一个简单的 Rust Hello World 程序",
    "project_id": "hello_world_demo"
  }')

SESSION_ID=$(echo $RESPONSE | jq -r '.data.session_id')
echo "会话ID: $SESSION_ID"
```

### 步骤 4: 监听SSE响应
```bash
# 开启新终端，监听SSE流
curl -X GET "http://localhost:8087/agent/progress/$SESSION_ID" \
  -H "Accept: text/event-stream"
```

### 步骤 5: 验证项目创建
```bash
# 检查项目目录是否创建
ls -la my_projects/hello_world_demo
```

### 步骤 6: 测试反向代理
```bash
# 测试代理功能
curl --location "http://localhost:8087/proxy/8080/status"
```

通过以上步骤，您已经完成了从部署到使用 RCoder 服务的完整流程。现在您可以开始探索更多高级功能和配置选项。
