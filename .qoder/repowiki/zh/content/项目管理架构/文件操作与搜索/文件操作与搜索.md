## 简介

本项目提供了一套完整的文件操作与搜索功能，支持跨文件的全局文本搜索、Git仓库状态管理、冲突检测和分支合并处理。系统通过`search`模块实现高效的文本搜索，利用`git_store`封装Git操作，并通过`worktree_store`管理文件树状态。各组件协同工作，确保文件状态的一致性和操作的高效性。

## 核心组件分析

系统主要由以下几个核心组件构成：`search`模块负责文本搜索功能，`git_store`管理Git仓库状态，`conflict_set`处理合并冲突，`git_traversal`实现Git状态遍历，`worktree_store`管理工作树结构。这些组件通过清晰的接口和事件机制进行通信，形成了一个高效、可维护的文件操作与搜索系统。

### 搜索算法实现

搜索算法通过`search`方法实现，支持在缓冲区快照中进行全文搜索。对于文本搜索，使用Aho-Corasick算法进行多模式匹配；对于正则表达式搜索，使用fancy-regex库。搜索过程中会定期调用`yield_now().await`以避免阻塞事件循环。

```mermaid
flowchart TD
Start([开始搜索]) --> CheckEmpty["检查查询是否为空"]
CheckEmpty --> |是| ReturnEmpty["返回空结果"]
CheckEmpty --> |否| MatchType["判断搜索类型"]
MatchType --> |文本搜索| AhoCorasick["使用Aho-Corasick算法匹配"]
MatchType --> |正则搜索| Regex["使用正则表达式匹配"]
AhoCorasick --> Yield["定期yield避免阻塞"]
Regex --> Yield
Yield --> Collect["收集匹配结果"]
Collect --> ReturnResult["返回结果列表"]
```

## 性能优化建议

### 索引缓存策略

为提高搜索性能，建议实现索引缓存机制。可以将常用的搜索结果缓存起来，避免重复计算。对于频繁访问的文件，可以预先加载其内容到内存中。

```mermaid
flowchart LR
FileChange([文件变化]) --> UpdateIndex["更新索引"]
UpdateIndex --> CacheIndex["缓存索引"]
CacheIndex --> SearchRequest["搜索请求"]
SearchRequest --> CheckCache["检查缓存"]
CheckCache --> |命中| ReturnCached["返回缓存结果"]
CheckCache --> |未命中| SearchFile["搜索文件"]
SearchFile --> UpdateCache["更新缓存"]
UpdateCache --> ReturnResult["返回结果"]
```

### 增量搜索策略

采用增量搜索策略，只搜索发生变化的文件，而不是扫描整个项目。通过监听文件系统事件，可以及时更新搜索索引，保持搜索结果的实时性。

```mermaid
flowchart TD
FileSystemEvent([文件系统事件]) --> FilterEvent["过滤事件"]
FilterEvent --> |文件创建/修改| UpdateIndex["更新索引"]
FilterEvent --> |文件删除| RemoveFromIndex["从索引移除"]
UpdateIndex --> SearchQuery["搜索查询"]
SearchQuery --> CheckIndex["检查索引"]
CheckIndex --> |存在| ReturnFromIndex["从索引返回"]
CheckIndex --> |不存在| FullSearch["完整搜索"]
FullSearch --> UpdateIndex
```
