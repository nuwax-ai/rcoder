## 关键结论

### ✅ MCP 初始化为阻塞根因已确认

| 验证项 | 结果 |
|--------|------|
| 禁用 MCP 后 | ❌ 不再超时 |
| 启用 MCP 后 | ✅ 100s 超时 |

### 性能退化仍存在
- 第 3 轮仍有性能退化（20s vs 3s）
- 原因：容器累积（45 个）导致资源竞争
- 但这只是**性能问题**，不是**阻塞问题**

---

## 建议

1. **MCP 工具预安装** - 消除下载延迟
2. **容器复用** - 减少累积
3. **并发限制** - 控制最大容器数
