## 关键发现

### 1. LocalSet 串行问题已排除
- 每个 Agent 容器**独立**到达 `new_session` 调用
- 不存在一个线程阻塞导致其他线程无法执行的情况

### 2. 问题定位到黑盒内部
```
Rust (可控)              TypeScript/Node 黑盒 (不可控)
     │                           │
new_session() ──────────→ claude-code-acp
     │                           │
   日志可见                    阻塞发生在这里
```

### 3. MCP 初始化是阻塞点
所有超时日志都显示：
```
MCP 服务器: trends-hub (mcp-proxy → npx → mcp-trends-hub)
```

---

## 根因分析

### 为什么 MCP 初始化会阻塞？

1. **npx 包下载**: `npx -y mcp-trends-hub@1.6.0` 需要下载包
2. **并发网络请求**: 45 个容器同时下载 npm 包
3. **网络带宽竞争**: 导致下载变慢或超时
4. **Node.js 启动开销**: 每个 MCP 工具都是一个 Node 进程

### 验证方法
```bash
# 检查容器内 npm 缓存
docker exec computer-agent-runner-mcp_r3_u1 \
  ls -la /root/.npm/_npx/
```

## 结论

| 问题假设 | 验证结果 |
|----------|----------|
| LocalSet 串行阻塞 | ❌ 已排除 |
| 内存/CPU 竞争 | ⚠️ 可能是次要因素 |
| MCP 初始化阻塞 | ✅ **确认为根因** |

---

## 优化建议

1. **预缓存 npm 包**: 在镜像中预装 MCP 工具
2. **MCP 初始化超时缩短**: 100s → 30s，快速失败
3. **MCP 工具精简**: 减少不必要的工具
4. **容器复用**: 避免重复初始化 MCP
