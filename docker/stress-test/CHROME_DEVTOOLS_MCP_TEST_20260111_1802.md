## 关键发现

### 🔴 本地代理 MCP 也会导致阻塞

即使 `chrome-devtools` 不需要下载任何包：
- 只是连接本地 `mcp-proxy convert http://127.0.0.1:18099`
- **仍然出现 110-120s 阻塞**

### 根因修正

| 之前假设 | 验证结果 |
|----------|----------|
| npx 下载包导致阻塞 | ❌ 不完全正确 |
| 任何 MCP 初始化都会阻塞 | ✅ **确认** |

---

## 新的根因分析

阻塞发生在 `claude-code-acp` (TypeScript) 内部的 MCP 初始化：
1. 不是网络下载问题
2. 是 MCP 工具**连接/初始化**本身的问题
3. 高并发时 MCP 初始化竞争资源

---

## 建议

1. **MCP 初始化超时缩短** - 100s → 30s
2. **MCP 工具精简** - 减少必要工具数量
3. **容器复用** - 避免重复初始化
4. **调查 claude-code-acp** 内部 MCP 初始化逻辑

## 复现脚本
`stress_test_chrome_mcp.sh`
