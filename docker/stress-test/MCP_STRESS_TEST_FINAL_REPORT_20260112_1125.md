## 关键发现

### 🔴 反直觉结果

**10 MCP 工具场景比 1 MCP 更稳定**

- 默认 MCP (1个): R3 全部超时 (HTTP=000)
- 10 MCP 工具: R3 全部成功

### 🔍 可能原因

1. **MCP 初始化时间差异**
   - 1 MCP: 快速初始化 → 容器快速创建 → 资源竞争激烈
   - 10 MCP: 慢速初始化 → 容器逐步创建 → 资源压力分散

2. **请求间隔效应**
   - 10 MCP 初始化 ~90s，自然产生请求间隔
   - 1 MCP 初始化 ~8s，请求密集导致资源耗尽

3. **mcp-proxy 特有问题**
   - 默认配置使用 mcp-proxy → 可能存在连接复用/锁定问题
   - 10 MCP 使用独立进程 → 资源隔离更好

## 建议

### 生产环境

| 配置项 | 建议值 | 原因 |
|--------|--------|------|
| MCP 数量 | 3-5 个 | 平衡功能和性能 |
| 并发限制 | ≤ 10 | 避免资源耗尽 |
| 容器 TTL | 缩短 | 及时释放资源 |
| 轮次间隔 | ≥ 10s | 给系统恢复时间 |

### 调查方向

1. **mcp-proxy 连接问题**: 调查 R3 崩溃是否与 mcp-proxy 有关
2. **容器清理策略**: 考虑在高负载时主动清理旧容器
3. **资源监控**: 添加内存/CPU 监控告警
